## 1. Manager Summary

**Problem:** The two-signal classification architecture (Issue #42) is working correctly,
but the director signal is nearly silent for tradition Satellite categories (Giallo,
Blaxploitation, American Exploitation, Pinku Eiga, etc.). Director lists in
`lib/constants.py → SATELLITE_ROUTING_RULES` average 5–8 directors per tradition category
when 15–20+ are needed for reliable signal coverage. The result: structural triangulation
(country + genre + decade) does 27% of all classifications; director identity does 5.1%.
The architecture is capable of doing better — the data is not there to activate it.

**Impact if unfixed:**
- 212 films classified via structural_signal alone with no director confirmation
- 68 films flagged as `review_flagged` (ambiguous structural match, no director to
  disambiguate) — these need manual review instead of auto-classifying
- The SORTING_DATABASE (486 manual entries) remains the primary routing mechanism for
  tradition categories, which is expensive to maintain and scales poorly
- Lookup retirement (removing redundant SORTING_DATABASE pins) cannot proceed safely
  without higher director signal coverage to replace them

**Risk if fixed wrong:**
- False-positive director matches could route mainstream films into Satellite categories
  (e.g. a director with a common name matching a listed exploitation director)
- Duplicate director entries between categories could cause routing conflicts
- Directors who are in Core whitelist, if added to Satellite lists, could confuse
  integrate_signals() Priority #5/#6 logic

**Estimated effort:** 2–3 days. Research is the bottleneck — each category requires
consulting Wikipedia, published filmographies, and the existing SATELLITE_CATEGORIES.md
spec to confirm domain grounding. Code changes are minimal (constants.py additions only).

## 3. Root Cause Analysis

### RC-1: Director lists are too sparse for signal reliability

**Location:** `lib/constants.py` → `SATELLITE_ROUTING_RULES` → `directors` lists per category

**Mechanism:** `score_director()` in `lib/signals.py` checks `DIRECTOR_REGISTRY`, which is
built directly from `SATELLITE_ROUTING_RULES['directors']` lists. A director not in the
list produces no `DirectorMatch`. For tradition categories, most canonical directors from
the published scholarship are absent. Film by `massimo dallamano` (Giallo Core director,
4 films in the tradition): not in the list, produces no director signal. His films route
via `structural_signal` (Italy + genre + decade) if they match, or `review_flagged` if
the structural match is ambiguous. The director signal that should confirm and disambiguate
is silent.

**Scale:** ~100–130 canonical Satellite directors missing across all tradition categories.

### RC-2 (hypothesis only — requires separate probe): Core director name normalisation

**Location:** `lib/core_directors.py` → `is_core_director()` + `lib/signals.py` → `score_director()`

**Mechanism (unconfirmed):** In the bypass test, 93 films with explicit Core destinations
route to Satellite via `structural_signal` rather than `director_signal`. If `score_director()`
were finding the Core director, `integrate_signals()` would produce `director_signal` (Core)
or `both_agree`. The fact that `structural_signal` fires instead suggests a name normalisation
mismatch: the director name in the manifest CSV (from filename parser / OMDb) may not match
the format expected by `CoreDirectorDatabase.is_core_director()`.

**This is NOT in scope for Issue #44.** It requires a targeted probe before speccing:
```bash
python3 -c "
```

**Gate impact:** Adding directors can only ADD routing paths, never remove them. A film
that currently routes via `structural_signal` may now route via `director_signal` or
`both_agree` — same destination, different reason code. No existing correct classifications
can regress as a result of additions only.

**Exception:** If a director name substring matches an existing title word in an edge case.
The director matching logic in `score_director()` uses `_director_matches()` which does
substring matching for multi-word names and whole-token matching for single words. Verify
each new entry does not accidentally shadow a known SORTING_DATABASE film title.

**Downstream consumers of changed output:**
- `scripts/reaudit.py` — reads `reason` column. New reason codes for films that previously
  got `structural_signal` will now show `both_agree` or `director_signal`. Reaudit
  confirmation count should hold or improve; no wrong_tier regressions expected.
- `output/lookup_coverage.csv` — AGREE count will rise as director signals confirm
  more films that the structural signal also catches.
- `scripts/audit_lookup_coverage.py` — AGREE/DISAGREE/UNSORTED distribution will shift
  toward AGREE for tradition categories with new directors.
- `dashboard.py` Signal Quality panel — `both_agree` and `director_signal` counts will rise.

## 5. Proposed Fix

### Fix Description

**This is a literature review task, not a manual curation task.**

The two-signal architecture (`docs/architecture/TWO_SIGNAL_ARCHITECTURE.md`) establishes
that structural coordinates (country+decade+genre) identify which scholarly field to
consult, and published scholarship within that field contains director rosters that
populate Signal 1 (Director Identity). The methodology:

1. For each tradition category, identify 1–2 authoritative published sources
   (monograph, edited volume, or academic filmography)
2. Extract the director roster from those sources
3. Apply the SATELLITE_DEPTH §3 quality gates to each candidate
4. Add verified entries to `SATELLITE_ROUTING_RULES[category]['directors']` in
   `lib/constants.py`

This is a pure data addition — no logic changes, no new files, no new functions.
The existing `_build_director_registry()` and `score_director()` machinery handles
new entries automatically.

### Per-Director Inclusion Gate (SATELLITE_DEPTH §3)

Every proposed director must pass ALL five checks before addition. This is the quality
standard — not every director who worked in a country/decade qualifies.

- [ ] **Sustained body of work** — Multiple films in the tradition across multiple years.
  A director with 2–3 films is a practitioner; 8+ films across a career indicates commitment.
  Minimum: 3 films in the tradition. (SATELLITE_DEPTH §3, Criterion 1)
- [ ] **Formal distinctiveness** — A recognisable visual, narrative, or thematic language
  specific to their work in this tradition. Not just "good at the genre" but "their way of
  doing the genre is identifiable." (SATELLITE_DEPTH §3, Criterion 2)
- [ ] **Scholarship citation** — Director appears in at least one published source documenting
  the movement: monograph, Wikipedia category page, BFI list, or filmography collection.
  (CLAUDE.md Rule 4 — Domain Grounding)
- [ ] **Not in Core whitelist** — `CoreDirectorDatabase.is_core_director()` returns False.
  Core directors route via Priority #6 in `integrate_signals()`. (Issue #25)
- [ ] **Active period within decade bounds** — The director's work in this tradition falls
  within the category's documented decade range. A 2010s director cannot be added to a
  1960s–1980s category. (CLAUDE.md §4 — decade-bounding; MARGINS_AND_TEXTURE §2)
